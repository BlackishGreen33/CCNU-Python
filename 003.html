实现球拍放在下方

球拍位置

要达成这个要求，首先我们需要先将球拍从竖向转为横向，也就是改变 racket 类的宽高属性。 

# racket.py (修改默认宽高)
rktw = 100
rkth = 10

# pong.py (修改实例化传参的值)
rkt = racket(CRKT, (100, 10), 220 / FPS, "img/pingpongbat.png")

接下来就需要改变球拍一开始放置的位置。

# pong.py
rkt.rect.x = 0
rkt.rect.y = HEIGHT - 10

同时别忘了 racket 类中有个 reset 函数，它在游戏重新开始时负责重置球拍的位置，所以也需要进行修改。 

# racket.py
def reset(self):
    self.rect.y = 630 # 视窗高度(640) - 球拍厚度(10)

这样看似就结束了，但实际运行游戏时就会发现其实还存在很多逻辑问题。例如说球碰到原本的视窗左侧时应当改为反弹、游戏成功与失败判定的边界线应该改至下方等，接下来我们就来继续处里一下这些 bugs。

球的反弹

依据要求，这个游戏中的球只有两种情况下会反弹，一是碰到上、左、右视窗边界时，二是碰到球拍时。那我们先来处里一下边界的反弹，这部份是在 ball 类的 update 函数中实现的。 

# ball.py
def update(self):
    # 上边界处理
    if self.rect.y < 0:
        self.speedy = -self.speedy
    # 左右边界处理
    elif (
        (self.rect.x + self.width) > self.WIDTH
    ) or self.rect.x < 0:
        self.speedx = -self.speedx

# pong.py
if bball.rect.y + bball.rect.height > HEIGHT - rkt.rect.height:  # 下边界

控制方式改为鼠标移动或者左右方向键

这部份想说两种操作方式都不难，干脆就都做了。

鼠标移动球拍

这边做个动作拆解，也就是说要先监听并获取到鼠标当前的位置，接着改变球拍的位置。 

# pong.py
# 未暂停且未结束的情况下才处理移动
if (not ispause) and (not isfail):
    # 监听鼠标移动事件
    if event.type == pygame.MOUSEMOTION:
        rkt.rect.x = event.pos[0] - rkt.rktw // 2 # 根据鼠标的水平位置调整球拍的位置

那可能会想说为什么要 - rkt.rktw // 2，也就是减去一半的球拍宽度，这是为了让当前鼠标位置保持在球拍中间，不然玩家操作起来感觉会怪怪的。

左右方向键移动球拍

最后我们来实现左右方向键移动的功能。那我们一开始要获取到左右方向的触发就必须先 import 一下 K_a 和 K_d (我这边是用 A 和 D 键操作啦，毕竟原代码是用 W 和 S 键，喜欢用左右方向键的也可以 import 一下 K_LEFT 和 K_RIGHT)。 

# pong.py
from pygame.locals import K_SPACE, K_a, K_d

接着我们移动前要先检查球拍是否可以向左移动，确保球拍在向左移动 rkt.rkstep 个单位后不会超出屏幕左边界，并且检查键盘上的 A 键是否被按下，而后右移动的判定同理。 

# pong.py
# 未暂停且未结束的情况下才处理移动
if (not ispause) and (not isfail):
    # ... [鼠标移动球拍] ...
    if (rkt.rect.x - rkt.rkstep >= 0) and pygame.key.get_pressed()[K_a]:
        rkt.rect.x = rkt.rect.x - rkt.rkstep
    if (
        rkt.rect.x + rkt.rktw + rkt.rkstep <= WIDTH
    ) and pygame.key.get_pressed()[K_d]:
        rkt.rect.x = rkt.rect.x + rkt.rkstep

于是我们游戏的修改就大功告成🎉。
