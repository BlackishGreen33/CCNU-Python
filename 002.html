Python 中在持久化对象时使用 JSON 和 Pickle 有什么差异 ？

在 Python 中，JSON（JavaScript Object Notation）和 Pickle 是两种常用的持久化对象的方式。

特点比较

它们之间主要有以下几个差异：

序列化格式方面：

  • JSON 是一种文本格式，而 Pickle 是一种二进制格式。
  • JSON 使用可读性较好的纯文本表示对象，而 Pickle 使用 Python 特定的二进制格式。

可读性方面：

  • JSON 由于是文本格式，所以序列化后的 JSON 数据是可读的，并且可以被其他编程语言解析。
  • Pickle 生成的二进制数据不可读，只能由 Python 解析。

数据类型支持方面：

  • JSON 可以序列化基本的数据类型（如字符串、数字、布尔值）、列表、字典和嵌套结构的组合。但是，JSON 不支持所有的 Python 对象类型，例如，它不能直接序列化自定义类的实例。
  • Pickle 能够序列化几乎所有的 Python 对象，包括自定义类的实例、函数、模块等。

安全性方面：

  • JSON 由于是文本格式，它相对较安全，不会执行任意代码。
  • Pickle 是一种 Python 特定的格式，反序列化 Pickle 数据时存在风险，因为恶意 Pickle 数据可以包含可执行的代码。

跨平台兼容性方面：

  • JSON 是一种通用的数据交换格式，它在不同编程语言和平台之间具有良好的兼容性。
  • Pickle 是 Python 特有的格式，不适用于与其他编程语言交互。

小结

如果需要在 Python 中持久化简单的数据结构或与其他编程语言交互，JSON 是一个较好的选择。而对于复杂的 Python 对象，或者需要保留对象的状态和行为，以及在 Python 环境中使用，Pickle 是更合适的选项。在使用 Pickle 时应注意安全问题，只反序列化可信任的 Pickle 数据。

Python 如何实现类似 C++ STL 中模板的机制，举例说明。

在 Python 中，没有直接对应 C++ STL 中的模板机制，因为 Python 是一种动态类型语言，它的变量类型是在运行时确定的。不过，Python 提供了一些机制，如泛型编程和函数重载，可以实现类似于 C++ STL 中模板的一些功能。

泛型编程

Python 中的泛型编程可以通过使用类型提示和类型变量来实现。类型提示是一种在变量、函数参数、函数返回值等位置指定类型的语法。通过使用类型变量，可以编写通用的函数和类，以处理多种类型的对象。例如： 

from typing import List, T

def print_list(lst: List[T]) -> None:
    for item in lst:
        print(item)

# 使用泛型函数打印不同类型的列表
print_list([1, 2, 3])  # 打印整数列表
print_list(['a', 'b', 'c'])  # 打印字符串列表

通过类型提示和类型变量，我们可以编写一次通用的函数，并在不同类型的列表上调用它。

函数重载

Python 中没有像 C++ 那样的静态函数重载机制，但可以通过使用默认参数值和类型检查来模拟函数重载。例如： 

def add(a: int, b: int) -> int:
    return a + b

def add(a: str, b: str) -> str:
    return a + b

# 调用不同版本的 add 函数
print(add(2, 3))  # 输出 5
print(add('Hello', ' World'))  # 输出 'Hello World'

通过根据参数类型来选择不同的函数实现，我们可以实现类似于函数重载的效果。

需要注意一点是，这种方法仅在运行时进行类型检查，并不会在编译时捕获类型错误。由于 Python 的动态特性意味着函数的行为可以在运行时动态改变，因此需要谨慎使用这些技术，以避免引入不必要的复杂性和错误。
